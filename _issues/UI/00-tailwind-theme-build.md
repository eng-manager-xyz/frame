# UI-00 · Tailwind theme and build

## Outcome

Web SSR/hydration and desktop CSR consume one shadcn-inspired theme generated by a pinned Tailwind CSS v4 CLI. Release builds minify it, source detection covers Rust views, and CI rejects stale or oversized output.

## Legacy replacement

- Replace the web `STYLE` constant and desktop `app.css`/Trunk CSS asset with `frame_ui::STYLESHEET` and `UiStyles`.
- Replace duplicated local color/spacing rules with semantic Tailwind tokens and scoped product-layout utilities.

## Acceptance criteria

- [x] Root `package.json` and lockfile pin Tailwind and its CLI.
- [x] `crates/ui/styles/tailwind.css` declares explicit Rust source paths, semantic light/dark tokens, reduced-motion behavior, and forced-color compatibility.
- [x] The official web and desktop build scripts install from the lockfile when needed and run the minified CSS build before Trunk.
- [x] `check-ui-styles.py` proves deterministic output, resolved imports, token presence, and the 96 KiB budget.
- [x] Both applications embed the same generated stylesheet and no longer load their old CSS implementations.
