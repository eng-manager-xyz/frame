# 06 · Define shared Rust domain types, stable IDs, errors, and versioned API contracts

## Outcome

Desktop, UI, Worker, and media services exchange one versioned vocabulary with validated state transitions and stable wire compatibility.

## Current Cap reference

Cap types are distributed across Rust crates, TypeScript domain packages, API contracts, database schema, Tauri commands, and media-server payloads. The Frame scaffold only contains a minimal Video state and ObjectKey.

Reference snapshot: `CapSoftware/Cap@6ba69561ac86b8efdb17616d6727f9638015546b`.

## Dependencies

[#01](./01-p0-migration-charter-parity-slos.md), [#05](./05-p1-workspace-boundaries-policy.md)

## Scope

Inventory and define opaque IDs, tenant context, users, organizations, videos, uploads, objects, edits, jobs, progress, errors, pagination, timestamps, money/quotas, and auth claims. Establish versioning and unknown-field/enum behavior.

### Out of scope

Database queries, endpoint implementations, and UI components are implemented in later issues.

## Deliverables

- [ ] Shared domain and contract crates with strict constructors and Serde wire fixtures.
- [ ] Documented lifecycle state machines for recordings, uploads, media jobs, invitations, and deletions.
- [ ] A stable error envelope with public codes, retryability, correlation IDs, and redacted internal causes.
- [ ] Compatibility policy for additive changes, deprecation, schema versions, and unsupported clients.
- [ ] Golden JSON fixtures consumed by Rust, TypeScript compatibility tests during transition, and API documentation.

## Acceptance criteria

- [ ] Invalid IDs, object keys, state transitions, sizes, timestamps, and tenant crossings cannot be represented or are rejected at boundaries.
- [ ] Serialization tests freeze field names, enum values, nullability, and error codes.
- [ ] Older supported clients safely ignore approved additions and receive a version error for breaking changes.
- [ ] Types compile in Worker Wasm, native services, and Leptos targets without runtime leakage.
- [ ] Logs and Display implementations do not expose secrets, signed URLs, or sensitive payloads.

## Required test evidence

- Round-trip and property tests.
- Compatibility fixtures generated by current Cap and parsed by Frame.
- State-machine transition table with exhaustive tests.

## Risks and open questions

- Binding wire formats directly to storage rows makes future migrations brittle.
- Timestamp, bigint, null, and enum semantics differ across Rust, JavaScript, MySQL, SQLite, and JSON.

## Rollout and rollback

Introduce contracts behind versioned endpoints and Tauri commands. Retain old schemas until all supported clients are outside the deprecation window.
